# Contributing to AI-Hustler

首先，感谢您考虑为AI-Hustler项目做出贡献！🎉

## 🤝 如何贡献

### 🐛 报告Bug

如果您发现了bug，请通过以下方式报告：

1. 检查[Issues](https://github.com/your-username/ai-hustler/issues)是否已存在相同问题
2. 创建新Issue，使用Bug报告模板
3. 提供详细的问题描述和复现步骤

### ✨ 提出新功能

有想法？我们欢迎：

1. 在[Discussions](https://github.com/your-username/ai-hustler/discussions)发起讨论
2. 获得社区反馈
3. 创建Feature Request Issue

### 🔧 代码贡献

#### 开发环境设置

```bash
# 1. Fork项目
# 2. 克隆您的fork
git clone https://github.com/your-username/ai-hustler.git
cd ai-hustler

# 3. 安装依赖
npm install

# 4. 创建功能分支
git checkout -b feature/AmazingFeature

# 5. 开发并测试
npm test

# 6. 提交更改
git commit -m "Add some AmazingFeature"

# 7. 推送到您的fork
git push origin feature/AmazingFeature

# 8. 创建Pull Request
```

#### 代码规范

- 使用ES6+语法
- 遵循现有代码风格
- 添加适当的注释
- 编写测试用例
- 更新相关文档

### 📚 文档改进

文档永远可以更好！您可以：

- 修复拼写错误
- 改进说明清晰度
- 添加使用示例
- 翻译文档
- 创建教程

## 📋 Pull Request流程

### 1. 创建PR前

- [ ] 代码通过所有测试
- [ ] 文档已更新
- [ ] 遵循代码规范
- [ ] 添加了适当的注释

### 2. PR模板

```markdown
## 📋 PR描述

简要描述本次更改

## 🎯 更改类型

- [ ] Bug修复
- [ ] 新功能
- [ ] 文档更新
- [ ] 性能优化
- [ ] 其他

## 🧪 测试

- [ ] 本地测试通过
- [ ] 添加了新测试
- [ ] 文档已更新

## 💡 其他信息

任何其他相关信息
```

### 3. 审核流程

1. 自动检查（CI/CD）
2. 代码审核
3. 功能测试
4. 合并到主分支

## 📜 行为准则

参与本项目即表示您同意遵守我们的行为准则：

- **友好** - 对所有人保持友好和尊重
- **耐心** - 耐心解答问题，帮助新手
- **建设性** - 提出建设性意见，避免负面批评
- **包容** - 欢迎不同背景和经验水平的贡献者
